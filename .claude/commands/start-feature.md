# /start-feature — Begin a new Kitchen-Layout-Visualizer feature

## Quick Steps

1. Read `AGENTS.md` at the repo root
2. Create `features/active/<feature-name>/` folder
3. Copy and fill `templates/01-product-spec-template.md` → `features/active/<name>/01-product-spec.md`
4. Copy and fill `templates/02-technical-spec-template.md` → `features/active/<name>/02-technical-spec.md`
5. Copy and fill `templates/03-implementation-plan-template.md` → `features/active/<name>/03-implementation-plan.md`
6. Identify relevant skills from the Skill Glossary in `AGENTS.md`
7. Read each relevant skill file in `skills/` — check `last_verified` date is recent
8. **Do NOT write any product code until `03-implementation-plan.md` is complete and all templates are filled**

## Rules
- If `features/active/<name>/` already exists, read the existing templates before proceeding
- If an eval case applies, link it: `evals/harness/<case>/prompt.md`
- If in doubt about architecture, read `openspec/specs/` for the relevant layer

## Full Playbook
See `commands/start-feature.md` for the complete workflow.

Now do the work described above.
